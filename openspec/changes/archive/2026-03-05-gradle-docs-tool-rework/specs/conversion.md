# Spec: HTML to Markdown Conversion

## Overview

HTML conversion happens inline during ZIP processing in `ContentExtractorService` — there
is no separate conversion pass and no intermediate raw HTML files on disk. Each ZIP entry
is converted in-memory and written directly to `converted/` as a `.md` file. This spec
defines the per-kind Jsoup strategies used during that inline conversion.

---

## Observed HTML Structure (Gradle 9.x)

Each docs section is generated by a different tool, so the HTML scaffolding varies
significantly. Selectors were verified against actual Gradle 9.4.0-rc-2 distribution files.

**File counts in `gradle-9.4.0-rc-2-docs.zip`**:

- `userguide/`: ~329 HTML files (AsciiDoc → Gradle site template)
- `dsl/`: ~340 HTML files (DocBook XSLT → Gradle site template, same outer template as userguide)
- `kotlin-dsl/`: ~8,445 HTML files (Dokka-generated)
- `javadoc/`: ~1,992 HTML files (JDK 21 Javadoc tool)
- `samples/`: ~48 description HTML files + 96 nested ZIPs (2 per sample: groovy-dsl + kotlin-dsl)
- `release-notes.html`: 1 file (custom Gradle HTML, no framework)

## HTML Conversion Strategy (per kind)

All kinds use Jsoup for parsing. The approach is:

1. Select the content root element from the full document.
2. Within the selected root, remove navigation/chrome elements.
3. Pass the remaining HTML to the Markdown converter.

No global pre-stripping of `nav`/`header`/`footer` tags — instead each kind strips only
the specific elements confirmed present in its template.

### USERGUIDE and SAMPLES

Both use the same Gradle site template (AsciiDoc-sourced pages for userguide;
AsciiDoc-sourced description pages for samples).

Content root selector (first match): `main.main-content`, `main`, `body`

Title: `doc.selectFirst("h1")?.text() ?: doc.title()`

Strip inside content root:

- `nav.docs-navigation` — left sidebar with User Manual / DSL Reference / Release Notes links

### DSL (Groovy DSL reference)

Uses the same Gradle site template outer wrapper as userguide. Generated by DocBook XSLT.

Content root selector (first match): `main.main-content`, `main`, `body`

Title: `doc.title()` (already clean: "Project - Gradle DSL Version X.Y.Z")

Strip inside content root:

- `nav.docs-navigation` — left sidebar with nav links and section anchors

Note: the `org.gradle.api.Project` DSL page is ~2000 lines. Property and method tables
are inside `div.chapter` within `main.main-content` — do not strip them.

### KOTLIN_DSL (Dokka-generated, stored under `kotlin-dsl/`)

Dokka generates a different structure from the main Gradle site template.

Content root selector (first match): `div#content`, `main`, `body`

Note: `div#content` contains `div.breadcrumbs` and `div.main-content`. `div#navigation-wrapper`
(the left nav panel) is a sibling of `div#content` in the DOM, NOT inside it, so selecting
`div#content` excludes it automatically.

Title: `doc.selectFirst("h1")?.text() ?: doc.title()`

Strip inside content root:

- `.breadcrumbs` — package path breadcrumbs (e.g. "gradle / org.gradle / BuildAdapter")
- `.footer` — Dokka version footer, if present inside `#content`

Keep: source links, member descriptions, `@since` / `@see` annotations, platform filter labels.

### JAVADOC (JDK 21 Javadoc tool)

JDK 21-style Javadoc wraps navigation in `<header role="banner" class="flex-header">` which
is a sibling of `<main role="main">`, not inside it.

Content root selector (first match): `main[role=main]`, `main`, `body`

Note: selecting `<main>` automatically excludes the `header.flex-header` navbar. No additional
stripping is needed for navigation elements.

Title: `doc.selectFirst("div.header h1, h1.title, h2.title, h1")?.text() ?: doc.title()`

Strip inside content root: nothing required (content is clean inside `<main>`).

Notes:

- `allclasses-index.html` and `index-all.html` are large (many classes). Convert fully —
  the table structure is useful for search.
- `div.header` inside `<main>` contains the class/interface title — keep it.
- Do NOT strip `div.sub-nav` or `div.sub-nav-list` — these contain member jump-links
  useful for navigation context.

### RELEASE_NOTES

Custom Gradle HTML, not generated by AsciiDoc, Dokka, or Javadoc.

Content root selector (first match): `div.container`, `body`

Title: `doc.selectFirst("h1")?.text() ?: doc.title()`

Strip inside content root:

- `.toc` — in-page table of contents

## Markdown Output Format

Use [flexmark-java](https://github.com/vsch/flexmark-java) (or equivalent) for HTML→Markdown
conversion applied after Jsoup content extraction. The Jsoup step removes scaffolding; the
Markdown converter handles the remaining HTML structure.

Output conventions:

- Headings preserved as `#`, `##`, etc.
- Code blocks preserved (Javadoc `<pre>`, AsciiDoc listing blocks)
- Tables preserved as GitHub-flavored Markdown tables where possible
- Links preserved as `[text](url)` — useful for cross-references
- Strip `<a>` anchors that are just section anchors with no text (e.g. `<a id="method-detail"></a>`)

## Non-HTML File Handling

| Extension                                        | Action                                        |
|--------------------------------------------------|-----------------------------------------------|
| `.md`                                            | Copy as-is                                    |
| `.gradle.kts`, `.gradle`                         | Copy as-is                                    |
| `.toml`, `.properties`                           | Copy as-is                                    |
| `.java`, `.kt`, `.groovy`                        | Copy as-is                                    |
| `README` (no extension)                          | Copy as-is                                    |
| `.txt`                                           | Copy as-is                                    |
| `.jar`, `.class`, `.png`, `.gif`, `.svg`, `.zip` | Skip (binary)                                 |
| anything else                                    | Copy if text-readable (UTF-8), skip if binary |

## Test Approach

Tests use **real HTML files** copied from an actual Gradle `-docs` distribution and stored
in `src/test/resources/docs-html-samples/`. These files may be slightly out of date but
must reflect the actual HTML structure of each kind.

### Test files to include in `src/test/resources/docs-html-samples/`

Copy from the extracted Gradle `-docs` distribution. Verified paths for Gradle 9.4.0:

| File path (under test resources)                         | Source in distribution                                        | Rationale                                                     |
|----------------------------------------------------------|---------------------------------------------------------------|---------------------------------------------------------------|
| `userguide/command_line_interface.html`                  | `docs/userguide/command_line_interface.html`                  | Representative userguide page with sections                   |
| `dsl/org.gradle.api.Project.html`                        | `docs/dsl/org.gradle.api.Project.html`                        | Large DSL type page (~2000 lines) with property/method tables |
| `kotlin-dsl/gradle/org.gradle/-build-adapter/index.html` | `docs/kotlin-dsl/gradle/org.gradle/-build-adapter/index.html` | Simple Dokka type page (avoids large `-project/index.html`)   |
| `javadoc/org/gradle/api/Project.html`                    | `docs/javadoc/org/gradle/api/Project.html`                    | Complex Javadoc page with JDK21-style navbar                  |
| `javadoc/allclasses-index.html`                          | `docs/javadoc/allclasses-index.html`                          | Javadoc all-classes index page                                |
| `samples/sample_building_java_applications.html`         | `docs/samples/sample_building_java_applications.html`         | Sample description page (same template as userguide)          |
| `release-notes.html`                                     | `docs/release-notes.html`                                     | Full release notes page (custom `div.container` structure)    |

Note: `kotlin-dsl/gradle/org.gradle/-project/index.html` is very large. Use a smaller
type page for tests (e.g. `-build-adapter/index.html`) to keep test resource size reasonable.

### HtmlConversionServiceTest

All tests load the real HTML file from test resources, run conversion, and assert on the output.

- **`userguideTitleExtracted`**: Converted userguide page has correct `# Title` as first heading
  (e.g. "Command-Line Interface" from `<h1>`).
- **`userguideNavSidebarStripped`**: Converted userguide page does not contain the
  `docs-navigation` sidebar content (e.g. does not contain "User Manual Home" or
  "Getting Started" nav links).
- **`userguideContentPreserved`**: Converted userguide page contains key CLI option descriptions
  (spot-check a known option like `--info`).
- **`dslPropertyTableConverted`**: Converted DSL Project page contains `dependencies` property
  in a readable form.
- **`dslNavSidebarStripped`**: Converted DSL page does not contain the `docs-navigation`
  sidebar content (e.g. does not contain "DSL Reference Home" nav link).
- **`kotlinDslBreadcrumbsStripped`**: Converted Kotlin DSL page does not contain breadcrumb
  text (e.g. does not contain the "gradle / org.gradle" breadcrumb path).
- **`kotlinDslMemberDescriptionsPreserved`**: Converted Kotlin DSL page contains meaningful
  API descriptions (e.g. type name in a heading).
- **`javadocNavbarStripped`**: Converted Javadoc page does not contain navbar text
  (e.g. does not contain "Skip navigation links" or "Overview" nav tab text).
- **`javadocMethodDetailPreserved`**: Converted Javadoc Project page contains method
  signatures or descriptions (e.g. "dependencies").
- **`javadocIndexPageConverts`**: `allclasses-index.html` converts without error and
  produces non-empty output containing class names.
- **`samplePageConverts`**: Sample description page converts and contains the sample title.
- **`releaseNotesConverts`**: Release notes page converts and contains version-specific content
  (e.g. a version number heading).
- **`releaseNotesTocStripped`**: Converted release notes do not contain the in-page TOC
  (i.e. no raw list of anchor links at the top).
- **`nonHtmlFileCopiedAsIs`**: A `.gradle.kts` file passed through conversion is unchanged.
- **`binaryFileSkipped`**: A `.png` file is not written to `converted/`.
- **`doneMarkerPreventsReConversion`**: Second `ensureConverted` call with marker present
  does not re-process any files.
