## ADDED Requirements

### Requirement: Include Interpreted Query in Response

The search tool SHALL include the Lucene-interpreted query string (as generated by the `Query.toString()` method) in its output.

#### Scenario: Valid Query Confirmation

- **WHEN** the user searches for `LANGUAGE`
- **THEN** the system SHALL prepend the results with the interpreted query, e.g., "Interpreted Query: contents:language contents_exact:language^5.0"

#### Scenario: Syntax Error Explanation

- **WHEN** the user searches for invalid syntax like `LANGUAGE =`
- **THEN** the system SHALL include a "Partial/Failed Query Interpretation" if possible, to help the user identify the location of the syntax error.
